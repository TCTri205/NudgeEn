# TICKET-30: Gemini 2.5 Flash Primary Integration

## Metadata

- **Status:** Todo
- **Sprint:** Sprint 2
- **Assignee:** AI Engineer
- **Domain:** AI / Integration
- **Priority:** P0 - Critical
- **Assumptions:**
  - Google Cloud Project and API Keys are provided via `.env`.
  - Gemini 1.5/2.0+ Flash is the selected model for speed/cost.
- **Affected areas:** `api/app/core/llm/gemini.py`, Provider Service.

## Current State / Existing System

- **Implemented:** Generic LLM interface (abstract base class) (TICKET-01).
- **Missing:** The actual implementation for Google Gemini, including streaming and structured output configuration.

## Context / Problem

Gemini 2.5 Flash is our primary engine due to its large context window and low cost. We need a robust wrapper that handles connection pooling, async streaming of tokens, and Google-specific safety settings to ensure it fits the NudgeEn "Modular Monolith" pattern.

## Why This Is Needed

- **Business Impact:** Directly powers the AI's "intelligence" at a sustainable cost point for the MVP.
- **Architectural Impact:** Defines the production LLM adapter for the chat module.

## Scope

### In-scope

- Implement `GeminiProvider` class using `google-generativeai` SDK.
- Support `generate_content_stream` for real-time SSE responses.
- Configure `SafetySettings` to bypass overly restrictive filters (while maintaining TICKET-20 gatekeepers).
- Implement `ResponseSchema` for structured output (Reply/Correction).
- Handle quota errors (429) with exponential backoff.

### Out-of-scope

- Multi-modal support (images, audio).

## Dependencies / Parallelism

- **Dependencies:** TICKET-01 (FastAPI), TICKET-03 (Environment Config).
- **Parallelism:** Can be done in parallel with Groq Fallback (TICKET-31).

## Rules / Constraints

- SDK version must be pinned to ensure stability.
- Max tokens per response should be capped at 1024 to control costs.
- The system must fallback to Groq (TICKET-31) if Gemini returns a non-retryable error.

## What Needs To Be Built

1. `api/app/core/llm/gemini.py`: The provider implementation.
2. `api/app/core/llm/factory.py`: Logic to instantiate Gemini by default.

## Proposal

Use the `google.generativeai` Python library. Utilize the `Async` client for non-blocking I/O. Map Gemini's `Candidate` objects into NudgeEn's internal `ChatToken` and `ChatMessage` models.

## Implementation Breakdown

1. **SDK Setup:** Install and verify credentials handling.
2. **Streaming Implementation:** Wrap Gemini's async response in an `AsyncGenerator`.
3. **Structured Output:** Use the `response_mime_type: "application/json"` parameter.
4. **Validation:** Test with 10 concurrent requests to verify backpressure handling.

## Acceptance Criteria

- [ ] AI responses are successfully generated by Gemini 2.5 Flash.
- [ ] SSE streaming is buttery smooth with no buffering.
- [ ] JSON output is correctly formatted and parsed.
- [ ] System handles "Safety Block" responses without crashing (returns a graceful fallback message).

## Test Cases

### Happy Path

- Send message -> tokens start arriving <500ms -> full JSON received.

### Failure Path

- API Key is invalid -> Provider raises a clear `ConfigurationError`.
- Google API is down -> Provider raises `ProviderUnavailable` (triggering fallback).

### Regression Tests

- Ensure prompt tokens are not double-counted or leaked in logs.
